Core Contradictions 04

CONTRADICTIONS

三大根本矛盾

01 / 8 模块框架 vs 叙事体

一句话逻辑假设全书有一个核心论点——小说有多重主题层叠。核心知识地图假设支柱并列递进——小说是叙事动力链。落地行动指南对小说几乎无用。

02 / 压缩策略的致命伤

逻辑种子找的是论点，不是叙事动力。章节蒸馏 500 Token，小说一章的关键细节就是这个量。关键段落直传只保留 20%，但小说的信息是弥散式的。

03 / 抓不住主线的深层原因

主线不是论点，是叙事动力。知识图谱的 Schema 为概念关系设计，不是叙事因果。3-Agent 分工割裂了叙事的整体性——架构师丢情感，实战派丢上下文，策展人丢过程。

核心洞察

对非虚构，蒸馏的是论点；对小说，蒸馏的是体验。论点可以压缩，体验必须复原。

Agent Redesign 08

3-AGENT REDESIGN

3-Agent 重新分工

Alpha → 叙事分析师

负责 M1 叙事内核 + M2 人物谱系 + M3 情节线索网。不再找论点，追踪人物欲望如何碰撞出情节。重点保留：人物动机的细节、情节转折的因果链、伏笔的埋设与回收。

Beta → 场景考古学家

负责 M4 关键场景复原 + M5 主题层叠 + M6 意象体系。不再设计行动闭环，而是复原关键场景。每个场景卡片：上下文 / 对话节选 / 人物内心 / 隐喻层 / 对全局的意义。追踪意象从首次出现到最终演变的完整轨迹。

Gamma → 策展人

负责 M7 金句与遗珠 + M8 叙事边界。金句必须附带场景上下文。遗珠重点：被主线遮蔽的支线、被结局覆盖的过程、沉默角色的故事。叙事边界：标注哪些细节一旦丢失，整部小说的解读就会偏移。

核心转变

旧：3-Agent 按功能拆解（结构 / 行动 / 策展）
新：3-Agent 按叙事层次拆解（动力链 / 场景层 / 边界层）
小说需要的是一个完整叙事被不同视角照亮，而不是被不同功能拆解。

Layers

STRATEGY UPGRADE

压缩策略 + 检索工具箱

压缩策略调整

  • 叙事种子 3000-5000 Token（比逻辑种子宽 60%）
  • 章节叙事摘要 1000-1500 Token（比非虚构宽 1 倍）
  • 强制保留：每章 2-3 段原文对话
  • 30-60 万字：增加 15-20 个场景卡片，每个保留原文 300-500 字
  • 关键场景不压缩，只标注叙事功能

Python 检索工具箱新增

  • search_scene.py — 按人物 / 情感 / 转折点检索关键场景
  • trace_arc.py — 追踪某角色的完整弧线（从欲望到结局）
  • find_motif.py — 追踪某意象 / 母题的演变轨迹
  • check_foreshadow.py — 验证伏笔埋设与回收的完整性
信息密度判定翻转：非虚构 — 20% 段落承载 80% 论据 → 精准命中。小说 — 100% 段落共同编织体验网 → 删除任何细节可能同时删掉伏笔线、人物侧写、一层隐喻。小说的信息密度分布是弥散式的，不是聚集式的。
