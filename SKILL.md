# 书籍深度蒸馏 · 认知图谱与智能体内参 (V3.0)

## 核心设计哲学

> **高维压缩，逻辑溯源，智能体检索。**
> 针对长文本，先提取"逻辑种子"以建立全局视野，再通过Python构建认知图谱实现结构化索引。
> Agent 不再被动接收信息，而是通过Bash调用Python检索脚本进行多跳推理。
> 视觉服务于逻辑：每一页 HTML 必须在 5 秒内传递一个核心决策点。

## 触发指令

```
/book-distillation <file_path> [style]
```

- `<file_path>`：必选。支持 .epub / .txt / .pdf / .md
- `[style]`：可选。"麦肯锡" / "极简" / "学术" / "科技感" / "暖色" / "安全对话"。未提供时智能匹配书籍情感基调

## 超长文本处理策略

### 文本分级与处理路径

```
文本量级判定
├─ ≤10万字 → 标准路径（全文直传3个Agent）
├─ 10-30万字 → 分层压缩路径（逻辑种子 + 章节摘要 + 关键段落直传）
├─ 30-60万字 → 三级蒸馏路径（逻辑种子 → 章节蒸馏 → 段落精炼 → Agent）
└─ >60万字 → Python认知图谱+智能体检索路径（逻辑种子 + FAISS向量索引 + NetworkX知识图谱 + Agent多跳推理检索）
```

### 分层压缩机制（10-60万字）

**第一层：逻辑种子（2000-3000 Token）**
- 全书骨架：章节标题 + 每章一句话核心论点
- 作者专属术语表（Unique Glossary）
- 论点递进线索图（Logic Thread）
- 核心隐喻与价值观（Meta-Value）

**第二层：章节蒸馏摘要（每章 500-800 Token）**
- 每章提取：核心论点 + 关键论据 + 作者原话（1-2句）
- 保留章节间的逻辑衔接标记
- 标记"高密度章节"，后续可按需展开

**第三层：关键段落直传（按需展开）**
- 仅对"高密度章节"展开原始段落
- 展开范围控制在总文本的 20% 以内

### Python认知图谱+智能体检索路径（>60万字）【V3核心升级】

**预处理（Phase 0中由主控Agent执行Python脚本）：**

1. **文本分段**：将全书按段落切分（每段≤500字），生成 `segments.json`
2. **向量语义层**：调用Python脚本生成段落向量索引（FAISS），语义标签
3. **结构图谱层**：调用Python脚本提取核心实体与关系，构建NetworkX知识图谱
4. **双索引对齐**：建立图谱节点/边与向量段落的双向映射指针

**Python检索工具箱（Agent通过Bash调用）：**

Agent 不再接收全文，而是通过Bash调用以下Python脚本进行多轮推理检索：

| 命令 | 适用场景 | 示例 |
|:---|:---|:---|
| `python /tmp/book_rag/search.py "查询内容" 5` | 语义相似度检索 | 查找"对比说明法的操作步骤" |
| `python /tmp/book_rag/graph.py --entity "安全感" --depth 2` | 图谱结构检索 | 查找"安全感"相关的实体和关系 |
| `python /tmp/book_rag/multihop.py "沉默" "共同目的" 3` | 多跳推理 | 从"沉默"到"共同目的"的推理链 |

**Agentic Loop 检索模式：**
```
Agent评估当前信息缺口
  → 通过Bash调用Python检索脚本
    → 评估返回结果是否足够
      → 若不足：换工具或修改查询再次检索
      → 若充足：生成最终内容
```

### 检索脚本接口规范

**search.py — 语义检索：**
```bash

python /tmp/book_rag/search.py "<查询文本>" [top_k] [--chapter "章节名"]

# 输出格式（JSON）
{
  "results": [
    {"segment_id": 42, "chapter": "第3章", "text": "...", "score": 0.87},
    ...
  ]
}
```

**graph.py — 图谱检索：**
```bash
# 用法
python /tmp/book_rag/graph.py --entity "安全感" [--relation "因果"] [--depth 2]

# 输出格式（JSON）
{
  "entity": "安全感",
  "related": [
    {"entity": "共同目的", "relation": "因果", "path": ["安全感", "相互尊重", "共同目的"], "evidence": ["段落ID: 42", "段落ID: 87"]},
    ...
  ]
}
```

**multihop.py — 多跳推理：**
```bash
# 用法
python /tmp/book_rag/multihop.py "起始实体" "目标实体" [max_hops]

# 输出格式（JSON）
{
  "start": "沉默",
  "end": "共同目的",
  "paths": [
    {"hops": 3, "path": ["沉默", "暴力", "安全感", "共同目的"], "evidence_segments": [12, 45, 89]},
    ...
  ]
}
```

## 工作流

### Phase 0: 逻辑种子与Python图谱构建

**目标：** 解决长文本导致的"Agent 疲劳"、"上下文丢失"与"关系割裂"。

1. **全局扫描**：识别书名、作者、核心隐喻及全书分析框架。
2. **文本分级**：判定字数量级，选择对应处理路径。
3. **逻辑种子生成**：提取 2000-3000 Token 的精华索引，包含：
   - **Unique Glossary**：作者定义的专属术语表
   - **Logic Thread**：论点随章节演进的递进线索图
   - **Meta-Value**：书籍底层价值观
   - **Density Map**：章节密度标注
4. **分层压缩**（>10万字时）：生成章节蒸馏摘要，标记高密度章节。
5. **Python认知图谱构建**（>60万字时）：
   - **Step 1**：部署工具到 `/tmp/book_rag/`：
     ```bash
     python ~/.claude/skills/book-distillation/tools/setup.py /tmp/book_rag
     ```
   - **Step 2**：将全书文本按段落切分，生成 `segments.json`：
     ```bash
     python /tmp/book_rag/segment_text.py /path/to/full_text.txt /tmp/book_rag/segments.json --max_len 500
     ```
   - **Step 3**：运行索引构建，生成FAISS向量索引 + NetworkX知识图谱：
     ```bash
     python /tmp/book_rag/build_index.py /tmp/book_rag/segments.json /tmp/book_rag
     ```
   - **Step 4**：验证索引完整性（至少1条跨章节实体关系链）
   - **Step 5**：验证检索工具可用：
     ```bash
     python /tmp/book_rag/search.py "测试查询" 3
     python /tmp/book_rag/graph.py --stats
     ```
6. **创建任务列表**。

**质量门控：**
- 逻辑种子必须包含至少3个作者专属术语 + 1条完整的递进线索
- >60万字时，知识图谱必须包含至少1条跨章节的实体关系链
- 否则重新提取或重新构建图谱

**降级策略：** 若Python依赖安装失败或图谱构建失败，降级为"主控Agent深度阅读路径"（主控Agent用Read逐章精读，手动构建逻辑图谱摘要，分发给子Agent）。

## 质量标准

1. **一句话逻辑**：精确到"不可替换"，揭示根本问题
2. **知识地图**：支柱数量反映原书结构，不硬性压缩，逻辑链完整
3. **颠覆性认知**：少而精，有原文证据支撑，与核心隐喻关联
4. **行动指南**：准备→执行→复盘完整，检索依据真实，场景展示完整循环，有错误vs理想对比
5. **金句**：形式多样，功能性优先，原话复刻不失真
6. **蒸馏清单**：核心实战工具保留框架，逻辑链无遗漏，省略理由透明
7. **行动地图**：关联同作者作品，有搜索关键词
8. **一致性**：所有模块围绕同一核心隐喻，术语统一
9. **导航线**：每页标注逻辑旅程位置
10. **HTML**：零依赖，浏览器直接打开，5秒/页可扫描
