# 书籍深度蒸馏 · 认知图谱与杂志风内参 (V4.0)

## 核心设计哲学

> **高维压缩，逻辑溯源，智能体检索。**
> 针对长文本，先提取"逻辑种子"以建立全局视野，再通过Python构建认知图谱实现结构化索引。
> Agent 不再被动接收信息，而是通过Bash调用Python检索脚本进行多跳推理。
> 视觉服务于逻辑：每一页 HTML 必须在 5 秒内传递一个核心决策点。

> **杂志风设计哲学：**
> 1. **克制优于炫技** — WebGL 背景只在 hero 页透出，平凡页几乎看不见
> 2. **结构优于装饰** — 不用阴影、不用浮动卡片、不用 padding box，一切信息靠字号 + 字体对比 + 网格留白
> 3. **字体分工法则** — 衬线（标题/重点/数字）→ 视觉重音；非衬线（正文）→ 信息密度；等宽（kicker/meta/foot）→ 装饰节奏
> 4. **图片第一公民** — 只裁底部，保证顶部和左右完整；网格用固定 height，不用 aspect-ratio
> 5. **节奏靠 hero 页** — hero 和 non-hero 交替，不累眼睛

## V4.0 核心升级

- **双轨产出**：路径A → Markdown 可生长知识库（面向 AI 问答 + 增量更新）；路径B → 杂志风 HTML PPT（面向人类阅读）
- **Wiki 知识库骨架**：Phase 0 自动生成 INDEX.md + concepts/ / actions/ / boundaries/ / entities/ 目录
- **[[双链]] 体系**：3-Agent 产出使用 Markdown 内部链接，概念间可追溯、可交叉引用
- **10 种杂志布局模板**：hero-cover / act-divider / big-numbers / text-image-split / image-grid / pipeline / question-page / big-quote / before-after / mixed-layout
- **模块8：边界约束**：新增红线清单 + 致命误区 + 适用边界，避免知识误用
- **5 套主题色**：墨水经典 / 靛蓝瓷 / 森林墨 / 牛皮纸 / 沙丘，T 键切换

## 超长文本处理策略

### 文本分级与处理路径

```
文本量级判定
├─ ≤10万字 → 标准路径（全文直传3个Agent）
├─ 10-30万字 → 分层压缩路径（逻辑种子 + 章节摘要 + 关键段落直传）
├─ 30-60万字 → 三级蒸馏路径（逻辑种子 → 章节蒸馏 → 段落精炼 → Agent）
└─ >60万字 → Python认知图谱+智能体检索路径（逻辑种子 + FAISS向量索引 + NetworkX知识图谱 + Agent多跳推理检索）
```

### 分层压缩机制（10-60万字）

**第一层：逻辑种子（2000-3000 Token）**
- 全书骨架：章节标题 + 每章一句话核心论点
- 作者专属术语表（Unique Glossary）
- 论点递进线索图（Logic Thread）
- 核心隐喻与价值观（Meta-Value）

**第二层：章节蒸馏摘要（每章 500-800 Token）**
- 每章提取：核心论点 + 关键论据 + 作者原话（1-2句）
- 保留章节间的逻辑衔接标记
- 标记"高密度章节"，后续可按需展开

**第三层：关键段落直传（按需展开）**
- 仅对"高密度章节"展开原始段落
- 展开范围控制在总文本的 20% 以内

### Python认知图谱+智能体检索路径（>60万字）【V3核心升级，V4保留】

**预处理（Phase 0中由主控Agent执行Python脚本）：**

1. **文本分段**：将全书按段落切分（每段≤500字），生成 `segments.json`
2. **向量语义层**：调用Python脚本生成段落向量索引（FAISS），语义标签
3. **结构图谱层**：调用Python脚本提取核心实体与关系，构建NetworkX知识图谱
4. **双索引对齐**：建立图谱节点/边与向量段落的双向映射指针

**Python检索工具箱（Agent通过Bash调用）：**

Agent 不再接收全文，而是通过Bash调用以下Python脚本进行多轮推理检索：

| 命令 | 适用场景 | 示例 |
|:---|:---|:---|
| `python /tmp/book_rag/search.py "查询内容" 5` | 语义相似度检索 | 查找"对比说明法的操作步骤" |
| `python /tmp/book_rag/graph.py --entity "安全感" --depth 2` | 图谱结构检索 | 查找"安全感"相关的实体和关系 |
| `python /tmp/book_rag/multihop.py "沉默" "共同目的" 3` | 多跳推理 | 从"沉默"到"共同目的"的推理链 |

**Agentic Loop 检索模式：**
```
Agent评估当前信息缺口
  → 通过Bash调用Python检索脚本
    → 评估返回结果是否足够
      → 若不足：换工具或修改查询再次检索
      → 若充足：生成最终内容
```

### 三层检索策略（V4.0核心升级）

```
┌─────────────────────────────────────────┐
│  第三层：多跳推理（深度）                  │
│  发现隐含因果关系、逻辑链条                │
│  工具：multihop.py + graph.py            │
├─────────────────────────────────────────┤
│  第二层：图谱关系验证（精度）              │
│  验证召回内容的关联性、扩展相关实体        │
│  工具：graph.py --entity --depth         │
├─────────────────────────────────────────┤
│  第一层：语义向量召回（广度）              │
│  用 embedding 相似度找到语义相关段落       │
│  工具：search.py（sentence-transformer） │
└─────────────────────────────────────────┘
```

**检索工作流：**
1. Agent 发起查询 → 先用 search.py 语义召回 top 10
2. 再用 graph.py 验证实体关联
3. 若需因果推理，用 multihop.py 探索路径
4. 综合三层结果，引用原文时标注来源段落ID

## 工作流

### Phase 0: 逻辑种子与Python图谱构建 + Wiki骨架

**目标：** 解决长文本导致的"Agent 疲劳"、"上下文丢失"与"关系割裂"；为双轨产出建立知识库骨架。

1. **全局扫描**：识别书名、作者、核心隐喻及全书分析框架。

1.1 **文本类型判定**：判断输入文本的体裁，后续全局适配。
   - **判定依据**：元数据体裁标签 / EPUB 分类 / 书名模式 / 章节命名风格 / 对话密度
   - 若为**小说/叙事类** → 全局启用"小说专项模式"（使用下方 3N 叙事种子替代 3 逻辑种子，4N 增强压缩替代 4 标准压缩，5N 叙事图谱替代 5 标准图谱）
   - 若为**非虚构类** → 继续标准路径

2. **文本分级**：判定字数量级，选择对应处理路径。

3. **逻辑种子生成**（非虚构类适用）：提取 2000-3000 Token 的精华索引，包含：
   - **Unique Glossary**：作者定义的专属术语表
   - **Logic Thread**：论点随章节演进的递进线索图
   - **Meta-Value**：书籍底层价值观
   - **Density Map**：章节密度标注
   - **Chapter Structure**：全书章节结构概览（每章标题 + 核心论点）
   - **Character System**：人物谱系（主要人物 + 关系 + 角色定位）
   - **Timeline**：核心情节 / 论点演进时间线

3N. **【小说专项】叙事种子生成**（当文本类型=小说时，替代第 3 步）
    提取 3000-5000 Token 的叙事索引（比非虚构种子宽60%），包含：
   - **核心冲突轴**：主矛盾起因、对抗双方、张力来源；次冲突线索；冲突的不可避性
   - **人物关系速查**：每个核心角色的欲望 + 人际关系 + 关键矛盾 + 弧线方向（上升/坠落/循环）+ 关键转折章节
   - **情节线索网**：主线起承转合 + N 条支线起承转合 + 各线索交汇点，标注章节范围
   - **关键意象表**：反复出现的象征物/场景/句式，首次出现章节 + 演变轨迹 + 指向的主题层
   - **叙事节奏图**：章节张力分布（起→铺垫→高潮1→低谷→高潮2→尾声），标注驱动事件

4. **分层压缩**（>10万字时）：生成章节蒸馏摘要，标记高密度章节。

4N. **【小说专项】增强压缩**（当文本类型=小说时，替代第 4 步）
   - 章节叙事摘要 **1000-1500 Token**（比非虚构宽 1 倍）
   - 每章强制保留 **2-3 段原文对话**
   - 标记"关键场景"（删除则故事塌陷的场景），不压缩，仅标注叙事功能
   - 场景卡片保底机制：如果删掉这个场景故事就塌了，那它必须保留
   - 30-60 万字时增加 15-20 个场景卡片，每个保留原文 300-500 字
5. **Python认知图谱构建**（>60万字时）：
   - **Step 1**：部署工具到 `/tmp/book_rag/`：
     ```bash
     python ~/.claude/skills/book-distillation/tools/setup.py /tmp/book_rag
     ```
   - **Step 2**：将全书文本按段落切分，生成 `segments.json`：
     ```bash
     python /tmp/book_rag/segment_text.py /path/to/full_text.txt /tmp/book_rag/segments.json --max_len 500
     ```
   - **Step 3**：运行索引构建，生成FAISS向量索引 + NetworkX知识图谱：
     ```bash
     python /tmp/book_rag/build_index.py /tmp/book_rag/segments.json /tmp/book_rag
     ```
   - **Step 4**：验证索引完整性（至少1条跨章节实体关系链）
   - **Step 5**：验证检索工具可用：
     ```bash
     python /tmp/book_rag/search.py "测试查询" 3
     python /tmp/book_rag/graph.py --stats
     ```

5N. **【小说专项】叙事知识图谱 Schema**（当文本类型=小说时，替代第 5 步的标准 Schema）
   - **节点类型**：
     | 节点 | 属性 |
     |:---|:---|
     | `character` | name / desire / arc_direction / key_chapters |
     | `event` | description / chapter / consequences / triggers |
     | `symbol` | name / first_appear / last_appear / evolution |
     | `location` | name / significance / associated_events |
     | `theme` | name / manifestations |
   - **边类型（叙事因果链）**：
     | 边 | 语义 | 用途 |
     |:---|:---|:---|
     | `--desires--→` | 人物欲望指向 | 角色动机追踪 |
     | `--obstructs--→` | 阻碍关系 | 冲突来源分析 |
     | `--triggers--→` | 触发事件 | 情节因果链 |
     | `--leads_to--→` | 因果推进 | 事件序列 |
     | `--foreshadows--→` | 伏笔→回收 | 跨章节结构（核心新增） |
     | `--mirrors--→` | 对称呼应 | 场景/人物的镜像关系 |
     | `--symbolizes--→` | 象征关系 | 意象→主题映射 |
     | `--transforms--→` | 演变轨迹 | 角色/意象变化追踪 |
   - **新增 Python 检索脚本**（>60万字时部署）：
     - `search_scene.py` — 按人物/情感/转折点检索关键场景
     - `trace_arc.py` — 追踪某角色的完整弧线（从欲望到结局）
     - `find_motif.py` — 追踪某意象/母题的演变轨迹
     - `check_foreshadow.py` — 验证伏笔埋设与回收的完整性

6. **Wiki 知识库骨架创建**（V4.0 新增）：
   - **Step 6**：创建 `{书名}-kb/` 目录结构，具体布局如下：
     ```
     {书名}-kb/
     ├── wiki/
     │   ├── INDEX.md              # 路由文件
     │   ├── concepts/             # 核心概念页面（模块2+3）
     │   │   └── example.md        # 每个概念一个独立文件
     │   ├── actions/              # 行动指南页面（模块4）
     │   │   └── example.md
     │   ├── boundaries/           # 边界约束页面（模块8）
     │   │   └── example.md
     │   ├── entities/             # 实体索引页面（人物/事件/章节）
     │   │   └── example.md
     │   ├── scenes/               # 【小说专项】场景卡片（模块4）
     │   │   └── example.md
     │   └── symbols/              # 【小说专项】意象体系（模块6）
     │       └── example.md
     └── output/                   # 最终交付物
     ```
   - **Step 7（新增）**：章节总结与人物谱系提取

## 降级策略

当Python依赖不可用或图谱构建失败时，自动降级：

```
Python可用？
├─ 是 → 完整GraphRAG路径（FAISS + NetworkX + Agent检索）
│       Phase 0: 完整逻辑种子 + 图谱索引 + Wiki骨架
│       Phase 1: 3-Agent 通过Bash调用Python检索
│       Phase 2: 图谱路径校验
│       Phase 3: 双轨产出（Markdown + HTML）
│
└─ 否 → 降级路径：主控Agent深度阅读
         ├─ 主控Agent用Read逐章精读，手动提取关键段落
         ├─ 手动构建逻辑图谱摘要（markdown格式）
         │   ├─ 核心概念列表 + 关系描述
         │   ├─ 跨章节逻辑流
         │   └─ 作者核心术语表
         ├─ 将精炼后的信息分发给子Agent
         └─ 子Agent无需Python工具，直接处理摘要
```

**降级路径下的 V4.0 特性支持：**
- Phase 0 Wiki 骨架创建：仍然可用（基于逻辑种子而非图谱）
- Phase 1 [[双链]] 体系：仍然可用（基于手动构建的概念关系）
- Phase 3 双轨产出：仍然可用（路径A Markdown 知识库 + 路径B 杂志风 HTML）
- Phase 4 质量检查：完全可用
- 唯一损失：图谱驱动的自动多跳推理和跨章节关系发现

**降级触发条件：**
- `pip install` 失败（网络受限或无权限）
- Python 运行环境缺失
- 文本过长导致内存不足（>100万字极端情况）
- FAISS/NetworkX 构建超时

## 质量标准

1. **一句话逻辑**：精确到"不可替换"，揭示根本问题
2. **知识地图**：支柱数量反映原书结构，不硬性压缩，逻辑链完整
3. **颠覆性认知**：少而精，有原文证据支撑，与核心隐喻关联
4. **行动指南**：准备→执行→复盘完整，检索依据真实，场景展示完整循环，有错误vs理想对比
5. **金句**：形式多样，功能性优先，原话复刻不失真
6. **蒸馏清单**：核心实战工具保留框架，逻辑链无遗漏，省略理由透明
7. **行动地图**：关联同作者作品，有搜索关键词
8. **边界约束**（V4.0）：红线清单清晰，致命误区准确，适用边界明确
9. **一致性**：所有模块围绕同一核心隐喻，术语统一，[[双链]] 可解析
10. **导航线**：每页标注逻辑旅程位置
11. **HTML**：零依赖，浏览器直接打开，5秒/页可扫描（V4.0 杂志风增加字体分工正确、主题交替合理、无 emoji 检查）
